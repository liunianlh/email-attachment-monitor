# 邮箱监控系统 — 产品设计文档

> 版本: v1.0 | 日期: 2026-06-08 | 状态: 待审阅

---

## 1. 产品概述

### 1.1 目标
自动监控网易企业邮箱，根据发件人规则，下载并处理邮件附件（Excel/CSV），清洗数据并校验合理性，通过后再执行用户自定义脚本，将结果推送至业务系统 API。

### 1.2 核心场景
- 用户A（供应商A）每天发日报 → 提取附件 → 清洗 → 校验 → 执行脚本A → 推送API
- 用户B（供应商B）每天发日报 → 提取附件 → 清洗 → 校验 → 执行脚本B → 推送API
- 规则数量不固定，需支持灵活扩展

### 1.3 部署环境
- 个人办公电脑（Windows / macOS / Linux 均支持）
- 每天下午 17:00 自动触发，开机即可
- 除 Python 3.10+ 外，零系统依赖

## 2. 技术栈

| 层 | 选型 | 备注 |
|---|------|------|
| 语言 | Python 3.10+ | 全平台统一 |
| 邮件拉取 | imaplib（标准库） + mail-parser | 纯 Python |
| Excel 处理 | pandas + openpyxl | 全平台一致 |
| 定时调度 | APScheduler | 进程内调度，不依赖 cron/launchd/任务计划程序 |
| Web 管理台 | Flask + Bootstrap 5（CDN） | 单 HTML 文件，浏览器访问 |
| 数据存储 | SQLite（标准库） | 零配置文件数据库 |
| 打包分发 | PyInstaller（可选） | 单文件 exe/app，无需安装 Python |

## 4. 规则模型

```yaml
规则:
  名称: "供应商日报处理"
  启用: true
  匹配条件:
    发件人:                          # 支持多个地址，命中任一即匹配
      - "supplier@company.com"
      - "supplier_backup@company.com"
    主题关键字: "日报"               # 可选
    附件匹配: "*.xlsx"               # 可选
  处理:
    保存路径: "./data/supplier/"
    Transform:
      编码: "auto"
      表头行: 1
      跳过行: [0]
      提取字段:
        - 列: "供应商名称"
        - 列: 2                     # 也支持按列号（从0开始）
      字段映射:
        "供应商名称": "supplier_name"
        "金额": "order_amount"
      清洗规则:
        金额字段: ["order_amount"]
        日期字段: ["order_date"]
    脚本路径: "./scripts/supplier_report.py"
    超时: 600                        # 秒
  输出:
    API端点: "https://api.example.com/orders"
    请求方式: "POST"
    认证头: "Bearer xxx"
```

**存储：** SQLite `rules` 表，通过 Web 管理台 CRUD。

## 5. 执行流水线

```
每天 17:00 APScheduler 触发
        │
        ▼
连接 IMAP，搜索 24h 内未读邮件
        │
        ▼
遍历每封新邮件
   ├── 匹配发件人列表 → 找到规则 → 未命中 → 跳过
   ├── 匹配主题关键字（可选）→ 不匹配 → 跳过
   ├── 下载匹配附件 → 保存到指定目录
        │
        ▼
   Transform 处理
   ├── 格式标准化（编码检测、表头识别）
   ├── 字段提取（按列名或列号）
   ├── 字段映射/重命名
   └── 数据清洗（去货币符号、统一日期格式、去空行）
        │
        ▼
   ★ 数据校验（基于模板）
   ├── 对照模板检查字段完整性
   ├── 对照模板检查数据类型
   ├── 对照模板检查值范围
   └── 不通过 → 记录日志，跳过脚本，跳过 API
        │
        ▼ 校验通过
   子进程调用用户脚本
   python3 ./scripts/xxx.py --data /tmp/data.csv --config /tmp/config.json
        │
        ▼
   读取脚本输出 → 推送 API
        │
        ▼
   记录执行日志（成功/失败/耗时/错误）
        │
        ▼
   标记邮件为已读
```

## 6. 数据校验引擎

用户提供一个「正确数据样本」Excel 模板，系统自动从中提取校验规则：

| 模板信息 | 校验行为 |
|---------|---------|
| 有哪些列、列名是什么 | 实际数据必须有这些列，列名必须匹配 |
| 每列的数据类型（数字/日期/文本） | 类型不匹配 → 校验失败 |
| 值的大致范围 | 明显异常值 → 校验失败 |

校验失败的结果：记录到日志，输出具体哪列、哪行、什么字段不符合模板定义。不执行脚本，不调用 API。

---

## 7. 脚本执行约定

- 系统以子进程方式调用用户脚本：`python3 <脚本路径> --data <数据文件> --config <规则配置JSON>`
- 脚本自行处理数据，退出码 0 = 成功，非 0 = 失败
- 脚本将结果写入约定输出路径
- 系统读取输出文件，按规则配置的 API 信息推送
- 超时默认 10 分钟，超时后强制终止并记录异常

## 9. 功能边界

### 系统做什么
- 每天 17:00 自动连接网易企业邮箱拉取新邮件
- 按发件人匹配规则，下载 Excel/CSV 附件
- Transform 清洗 → 模板校验 → 脚本执行 → API 推送
- 校验失败不执行脚本，记录错误详情
- Web 管理台管理规则、日志、模板

### 系统不做什么
- 不发送邮件
- 不修改邮箱内容（只读）
- 不处理非 Excel/CSV 附件
- 不提供移动端适配
- 不提供多用户/登录系统
